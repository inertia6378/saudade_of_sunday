Broken Authorization

Broken Authorization is when an applications functionality is broken or weak and attackers are able to bypass the authentication using session token flaws, brute forcing, no rate-limiting, password compromising using response manipulation or any other flaws which can give an attacker temporary or permanent access to the users identity.

Examples

  • OTP Bypass
  • Captcha Bypass
  • Common Passwords BruteForce
  • 2FA Bypass

Causes

  • Manipulated Response is interpreted by the application
  • No Rate Limit
  • Lack of Session Management

What Can Be Achieved

  • Temporary/Permanent Account Take Over
  • Modification of key user details
  • Usage of account to perform more malicious hacks

How to Fix/Prevent BA Issues

  • String Auth(JWT,SAML,OAuth)
  • Credential limits to ensure strength
  • Limit the Rate of interaction
  • Using Server side session management
